## `src.ydata_profiling.report.presentation.flavours.html.table.HTMLTable` · *class*

## Summary:
HTMLTable is a concrete implementation of the Table class that renders tabular data as HTML markup using Jinja2 templates.

## Description:
The HTMLTable class provides HTML-specific rendering capabilities for tabular data within the ydata-profiling report generation system. It inherits from the abstract Table class and implements the render() method to generate HTML table representations. This class serves as a bridge between structured tabular data and its HTML presentation, leveraging the Jinja2 templating engine to apply predefined formatting templates.

The class is designed to be used within the report generation pipeline where table data is prepared in the base Table class and then rendered into HTML format for web-based reporting. It follows the template method pattern by implementing the abstract render() interface defined in its parent class.

## State:
- Inherits all state from Table class including:
  - rows: Sequence containing the tabular data rows
  - style: Style configuration object for visual presentation
  - name: Optional[str] for human-readable identification
  - caption: Optional[str] for table descriptive text
- Inherits content dictionary from ItemRenderer parent class
- No additional instance attributes beyond those inherited

## Lifecycle:
- Creation: Instantiated with required rows and style parameters, similar to the base Table class
- Usage: Called during report generation when HTML table output is needed; render() method is invoked to produce HTML string
- Destruction: Managed by Python's garbage collection; no explicit cleanup required

## Raises:
- jinja2.exceptions.TemplateNotFound: If the "table.html" template file is not found in the Jinja2 template directory
- jinja2.exceptions.UndefinedError: If required variables from self.content are missing when rendering the template

## Example:
```python
from ydata_profiling.config import Style
from ydata_profiling.report.presentation.flavours.html.table import HTMLTable

# Create a table with sample data
style = Style()
rows = [["Name", "Age"], ["Alice", 30], ["Bob", 25]]
table = HTMLTable(rows=rows, style=style, name="Sample Data")

# Render the HTML table
html_output = table.render()
# Returns a string containing complete HTML table markup
```

### `src.ydata_profiling.report.presentation.flavours.html.table.HTMLTable.render` · *method*

## Summary:
Renders HTML table content by applying the table.html template to the instance's content dictionary.

## Description:
This method provides the concrete implementation of the abstract render() interface inherited from the base Table class. It serves as the HTML-specific rendering logic for table components within the ydata-profiling report generation system. During the report building process, this method is invoked when HTML-formatted table content needs to be generated from the structured data stored in self.content.

The method leverages the Jinja2 templating engine to apply the "table.html" template to the content dictionary, producing a complete HTML table representation. This approach separates the data structure (stored in self.content) from the presentation logic, enabling flexible and maintainable report generation.

## Args:
    None

## Returns:
    str: A string containing the fully-rendered HTML table markup generated by applying the table.html template to self.content

## Raises:
    jinja2.exceptions.TemplateNotFound: If the "table.html" template file is not found in the template directory
    jinja2.exceptions.UndefinedError: If required variables from self.content are missing when rendering the template

## State Changes:
    Attributes READ: self.content
    Attributes WRITTEN: None

## Constraints:
    Preconditions:
    - self.content must be a dictionary containing all required variables for the "table.html" template
    - The "table.html" template must exist in the Jinja2 environment
    - The HTMLTable instance must have been properly initialized with valid content (rows, style, etc.)

    Postconditions:
    - Returns a valid HTML string representation of the table
    - The returned string contains properly formatted HTML table markup

## Side Effects:
    None
