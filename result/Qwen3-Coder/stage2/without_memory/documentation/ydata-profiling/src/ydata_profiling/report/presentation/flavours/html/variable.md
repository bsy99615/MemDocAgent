## `src.ydata_profiling.report.presentation.flavours.html.variable.HTMLVariable` · *class*

## Summary:
HTMLVariable is a presentation layer class that renders variable information as HTML using Jinja2 templates.

## Description:
HTMLVariable is a concrete implementation of the Variable abstract class designed specifically for HTML presentation. It serves as a bridge between variable data structures and their HTML representation. This class is typically instantiated by the HTML presentation framework when rendering profiling reports in HTML format. The class leverages Jinja2 templating to generate HTML output from variable content.

## State:
- `content` (dict): Inherited from Renderable, contains the variable data including 'top', 'bottom', and 'ignore' keys. The 'top' and 'bottom' values are typically renderable objects that will be processed by the template.
- `item_type` (str): Inherited from ItemRenderer, set to "variable" indicating this is a variable-type item.
- `name` (str): Optional name property inherited from Renderable, stored in content dictionary.
- `anchor_id` (str): Optional anchor identifier inherited from Renderable, stored in content dictionary.
- `classes` (str): Optional CSS classes inherited from Renderable, stored in content dictionary.

## Lifecycle:
- Creation: Instantiated by the HTML presentation framework with appropriate content containing 'top' and 'bottom' renderable objects. The constructor inherits from the parent classes and sets up the content structure.
- Usage: Called by the HTML presentation system when rendering variable sections of reports. The `render()` method is invoked to generate HTML output.
- Destruction: No special cleanup required; relies on Python's garbage collection.
