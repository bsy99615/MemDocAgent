# `image.py`

## `src.ydata_profiling.report.presentation.flavours.html.image.HTMLImage` · *class*

## Summary:
Represents an HTML-formatted image element for report presentation that renders using a Jinja2 template.

## Description:
The HTMLImage class is a concrete implementation of the abstract Image class specifically designed for HTML report generation. It extends the base Image presentation element by implementing the render method to produce HTML markup using the diagram.html template. This class serves as a bridge between image data stored in the content dictionary and its HTML representation in generated reports.

This class is typically instantiated by the report generation system when creating HTML presentations that include image elements. It's part of the HTML flavour presentation layer that handles the rendering of report components in HTML format. The class is particularly useful for displaying charts, graphs, and other visualizations within HTML reports generated by ydata-profiling.

## State:
- Inherits all attributes from the parent Image class including:
  - image: str - Path or URL to the image resource
  - image_format: ImageType - Format of the image (PNG, JPEG, SVG, etc.)
  - alt: str - Alternative text for accessibility purposes
  - caption: Optional[str] - Optional caption text for the image
  - item_type: str - Fixed value "image" indicating the type of presentation element
  - content: dict - Dictionary containing all image metadata including image, image_format, alt, and caption
- The content dictionary is populated during object initialization with image metadata through the parent class constructor
- The content dictionary is used by the render method to populate the template variables

## Lifecycle:
- Creation: Instantiated with image path/URL, image format, and alternative text. Caption is optional. The parent Image class constructor validates parameters and populates the content dictionary.
- Usage: Called by the report generation system when rendering HTML reports containing image elements. The render() method is invoked to generate HTML output by rendering the diagram.html template with the content data.
- Destruction: Managed by Python's garbage collection when no longer referenced.

## Raises:
- ValueError: Inherited from parent Image class when the image parameter is None during initialization
- TemplateNotFound: May be raised by templates.template() if "diagram.html" template is not found
- UndefinedError: May be raised by Jinja2 template rendering if required variables are missing from self.content

## Example:
```python
from ydata_profiling.config import ImageType
from ydata_profiling.report.presentation.flavours.html.image import HTMLImage

# Create an HTML image with caption for inclusion in an HTML report
html_img = HTMLImage(
    image="path/to/chart.png",
    image_format=ImageType.PNG,
    alt="Distribution chart of age variable",
    caption="Figure 1: Age distribution"
)

# Render the HTML output for inclusion in an HTML report
html_output = html_img.render()
# Returns HTML string using the diagram.html template with the image content
# This HTML can be embedded directly into larger HTML documents
```

### `src.ydata_profiling.report.presentation.flavours.html.image.HTMLImage.render` · *method*

## Summary:
Renders an HTML representation of an image element using the diagram.html Jinja2 template.

## Description:
The render method generates HTML markup for an image element by processing the image's metadata through the "diagram.html" Jinja2 template. This method is part of the HTML presentation flavour implementation for image elements in the ydata-profiling report generation system.

During report generation, when HTML output is being produced, this method transforms image metadata (stored in self.content) into properly formatted HTML. The method leverages the Jinja2 templating system to apply the diagram.html template to the image's content dictionary.

This method exists as a separate implementation because it needs to handle the specific HTML rendering requirements while maintaining consistency with the base Image class interface, allowing different presentation flavours (HTML, JSON, etc.) to render the same underlying image data in their respective formats.

## Args:
    None

## Returns:
    str: HTML-formatted string representing the rendered image element

## Raises:
    None explicitly raised

## State Changes:
    Attributes READ: self.content (dictionary containing image metadata including image path, format, alt text, and optional caption)
    Attributes WRITTEN: None

## Constraints:
    Preconditions: 
    - self.content must be a dictionary containing valid keys for the diagram.html template
    - The diagram.html template must exist in the template directory
    - The parent Image class must have properly initialized the content attribute with image metadata
    
    Postconditions:
    - Returns a properly formatted HTML string for the image element
    - The returned string follows the expected HTML structure for diagram elements

## Side Effects:
    None
