# `render_file.py`

## `src.ydata_profiling.report.structure.variables.render_file.render_file` · *function*

## Summary
Generates HTML report template variables for file-type variables by extending path variable rendering with file-specific visualizations and metadata.

## Description
The `render_file` function extends the path variable rendering logic to include file-specific information and visualizations. It builds upon the template variables generated by `render_path` to add file size histograms and file timestamp frequency tables. This function is responsible for creating a complete report structure for file variables that includes both the standard path analysis and file-specific metadata.

The function is called during the report generation phase when processing file-type variables, specifically when the variable type has been identified as a file path. It integrates seamlessly with the existing path variable reporting infrastructure while adding file-specific analytical components.

## Args
    config (Settings): Configuration object containing report settings including:
        - `n_freq_table_max`: Maximum number of entries to display in frequency tables
        - `plot.image_format`: Format for generated plots (png, svg, etc.)
        - Other configuration parameters affecting report generation
    summary (dict): Dictionary containing file variable statistics including:
        - "varid": Variable identifier
        - "file_size": File size data for histogram generation
        - "histogram_file_size": Histogram data for file size visualization
        - "file_created_time": Created timestamp frequency data
        - "file_accessed_time": Accessed timestamp frequency data
        - "file_modified_time": Modified timestamp frequency data
        - "n": Total count of observations
        - Other keys required by the underlying rendering functions

## Returns
    dict: Template variables dictionary containing:
        - All template variables from `render_path` extended with file-specific components
        - A "File" tab in the bottom section containing file size histogram and timestamp frequency tables
        - Updated "var_type" field in the top section set to "File"
        - All other standard template variables for HTML report rendering

## Raises
    KeyError: When required keys are missing from the summary dictionary (e.g., "varid", "histogram_file_size", etc.)

## Constraints
    Preconditions:
        - The summary dictionary must contain "varid" key
        - The summary dictionary must contain "n" key with total observation count
        - If "file_size" key exists, "histogram_file_size" must also exist
        - If file timestamp keys exist, they must contain valid frequency data
        - Config must contain required settings for rendering

    Postconditions:
        - Returns a complete template variables dictionary ready for HTML rendering
        - The returned dictionary maintains the same structure as `render_path`
        - File-specific tabs are properly added to the bottom container
        - The var_type in the top section is set to "File"

## Side Effects
    None - This function is pure and does not modify external state or perform I/O operations

## Control Flow
```mermaid
flowchart TD
    A[Start render_file] --> B[Extract varid from summary]
    B --> C[Call render_path to get base template variables]
    C --> D[Set var_type to "File" in top section]
    D --> E[Get config settings n_freq_table_max and image_format]
    E --> F[Initialize empty file_tabs list]
    F --> G{file_size in summary?}
    G -->|Yes| H[Create Image component with file size histogram]
    H --> I[Append Image to file_tabs]
    G -->|No| J[Skip file size histogram]
    J --> K[Define file_dates mapping]
    K --> L[Iterate over file_dates items]
    L --> M{Current file_date_id in summary?}
    M -->|Yes| N[Create FrequencyTable for timestamp data]
    N --> O[Append FrequencyTable to file_tabs]
    M -->|No| P[Skip timestamp frequency table]
    P --> Q[Loop to next file date]
    Q --> R[Create Container with file_tabs as tabs]
    R --> S[Append file_tab to template_variables["bottom"]["content"]["items"]]
    S --> T[Return template_variables]
```
