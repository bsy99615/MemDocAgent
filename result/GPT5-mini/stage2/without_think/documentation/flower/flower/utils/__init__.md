## `flower.utils.__init__.abs_path` · *function*

## Summary:
Expand a leading tilde (~) and, if needed, resolve a relative path by joining it with a chosen working directory (prefer PWD environment variable, otherwise os.getcwd()).

## Description:
This utility normalizes the provided path string-like value in two steps, matching the implementation order:
1. Calls os.path.expanduser(path) to expand a leading tilde (~).
2. If the expanded value is not absolute (os.path.isabs returns False), obtain a working directory via os.environ.get('PWD') or, if that is falsy, os.getcwd(), and return os.path.join(cwd, expanded_value). If the expanded value is already absolute, it is returned as-is (the expansion result).

Known callers in the provided snapshot:
    - No call sites were discovered in the provided repository snapshot. The function exists as a general-purpose helper for code that needs a normalized path string before performing filesystem operations (e.g., CLI parsing, config loading).

Why this is a separate function:
    - Encapsulates a common normalization pattern (user expansion + relative-to-cwd resolution) so callers do not each reimplement the same behavior or the specific PWD vs os.getcwd() fallback.

## Args:
    path:
        - Expected: a string-like path value (typically str). The implementation forwards this value to os.path.expanduser, os.path.isabs, and os.path.join.
        - Practical recommendation: pass a str or an object compatible with os.fspath (os.PathLike). Passing None or other incompatible types will cause the underlying os.path functions to raise (e.g., TypeError).

## Returns:
    The value returned by the last os.path.* operation performed:
    - If os.path.expanduser(path) yields a value that os.path.isabs subsequently reports as absolute, that expanded value is returned.
    - Otherwise, returns os.path.join(cwd, expanded_value) where cwd is os.environ.get('PWD') if that returns a truthy value, otherwise the value returned by os.getcwd().
    - Typical return type is str when callers pass str. The function does not coerce types beyond what the underlying os.path functions do.
    - Note: The function does not guarantee the returned string is absolute if the chosen cwd (PWD or os.getcwd()) is itself not absolute.

## Raises:
    Exceptions are not caught; they propagate from the underlying calls:
    - TypeError (or similar) if os.path.expanduser/os.path.isabs/os.path.join reject the passed type (for example, passing None).
    - OSError (or subclass) if os.getcwd() fails and os.environ.get('PWD') is falsy — os.getcwd() is invoked only when PWD is falsy because of the "or" expression in the code.

## Constraints:
    Preconditions:
        - The caller should provide a value acceptable to os.path.expanduser (typically a str or os.PathLike).
        - If the caller relies on the returned value being absolute, ensure PWD (environment) is absolute or that os.getcwd() will succeed and return an absolute path.
    Postconditions:
        - The returned value equals:
            * os.path.expanduser(path) if that expanded value is absolute; otherwise
            * os.path.join(cwd, os.path.expanduser(path)) where cwd is os.environ.get('PWD') if truthy, else os.getcwd().
        - No filesystem modifications are performed by this function.

## Side Effects:
    - Reads the environment via os.environ.get('PWD').
    - May call os.getcwd(), which queries process state and can raise OSError.
    - Does not perform filesystem I/O (no file creation, deletion, or reads) and does not modify global variables.

## Control Flow:

    ExpandUser --> IsAbs{os.path.isabs(expanded) ?}
    IsAbs -- True --> ReturnExpanded[Return expanded]
    IsAbs -- False --> GetPWDOrCwd[cwd = os.environ.get('PWD') or os.getcwd()]

    ReturnExpanded --> End
    JoinPath --> End

## Examples:
1) Expand home shorthand:

    # out is the value of os.path.expanduser('~/myapp/config') (no filesystem ops performed)

    # out == os.path.join('/home/user/project', 'data.db')

## Summary:

## Description:

- None discovered in the provided file context. Typical callers in a web application are route-building utilities or HTTP handler code that need to assemble endpoint paths from a common prefix and a handler-specific suffix.

It enforces the responsibility of making sure the returned path always begins with a leading slash and that the prefix itself does not contribute extra leading/trailing slashes.

## Args:
    url (str):
        The URL suffix to append. This function does not alter url; it is concatenated verbatim after the normalized prefix. Expected to be a string. If it begins with a slash, the result will include that slash (potentially producing a double slash if prefix is empty).
    prefix (str):
        The prefix to place between the leading slash and the url. This is normalized by removing any leading and trailing slash characters (i.e., prefix.strip('/')). Expected to be a string. May be an empty string.

Interdependencies:
- Both arguments are expected to be strings. The function does not convert non-string inputs and will raise exceptions if they do not support the operations used (strip and string concatenation).

## Returns:

- prepend_url('/status', '') -> '//status'      (note: empty prefix yields a double leading slash)
- prepend_url('/status', '/api/') -> '/api/status'

## Raises:

    TypeError:
        If url is not a string and cannot be concatenated with the produced prefix string, Python will raise a TypeError during concatenation.

These exceptions are not explicitly raised by the function; they result from using .strip and plain string concatenation on values that do not support those operations.

## Constraints:

- Expectation that the caller understands whether url includes a leading slash to avoid unintended double slashes or missing separators.

- The prefix portion of the return value contains no leading or trailing '/' characters (because prefix.strip('/') is used).

## Side Effects:
