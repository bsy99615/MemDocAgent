# `locate_control.py`

## `folium.plugins.locate_control.LocateControl` · *class*

## Summary:
LocateControl is a Folium plugin Element that declares configuration for the leaflet-locatecontrol frontend library and registers the library's JavaScript and CSS resources with the hosting Figure via the JSCSSMixin.

## Description:
LocateControl is created when a map author wants to add the leaflet-locatecontrol plugin assets and an associated configuration object to a Folium Map/Figure. The class is a MacroElement that mixes in JSCSSMixin:

- It declares default_js and default_css class attributes that JSCSSMixin will register on the Figure header when the element is rendered.
- It stores an auto_start flag and an options dictionary generated by parse_options(**kwargs).

Important boundary and responsibilities (what this class guarantees):
- It guarantees that instances will have attributes _name, auto_start, and options after construction.
- It guarantees that the JS and CSS URLs listed in default_js and default_css are available on the class for JSCSSMixin to register at render time.
- It does not itself execute client-side locating logic; actual client-side behavior depends on the included frontend library and on any template content that consumes the stored options/flag.

## State:

        - Type: jinja2.Template
        - Value in source: an empty Template() placeholder.
        - Note: the source provides an empty template object. The presence of _template is required by MacroElement but the source does not populate it with rendering logic.
    - default_js
        - Type: list[tuple[str, str]]
        - Value: [("Control_locate_min_js", "https://cdnjs.cloudflare.com/ajax/libs/leaflet-locatecontrol/0.66.2/L.Control.Locate.min.js")]

## Summary:
Initializes the LocateControl plugin instance by performing base-class initialization, setting the plugin name, recording whether location tracking should start automatically, and normalizing any provided JavaScript/Leaflet option keywords into a dictionary stored on the instance.

## Description:
This constructor is invoked when a developer or user instantiates a LocateControl (e.g., LocateControl(auto_start=True, watch=True)) as part of building or adding a plugin to a folium Map. Typical callers:
- Application code or examples that add the locate control to a Map during map construction or plugin registration.
- Any factory/helper that constructs plugin objects prior to attaching them to a map.

Lifecycle context:
- Called at object creation time; the resulting instance is subsequently configured into the map rendering pipeline (the instance's serialized options are later used when generating the JavaScript for the control).

Rationale for having this logic in __init__:
- Keeps construction responsibilities localized: perform required base-class initialization, set intrinsic metadata (_name), record simple behavioral flags (auto_start), and normalize user-provided keyword options once on construction. Normalization is delegated to the shared parse_options helper to avoid duplicating key-formatting and None-filtering logic across plugins.

## Args:

        - If True, indicates that the client-side locate control should begin locating automatically after it is added to the map.
        - The constructor does not coerce or validate this value beyond assignment; callers should supply a boolean. Non-bool values will be stored as-is.
    **kwargs: keyword arguments mapping option names to values.

        - Any kwargs whose value is exactly None are removed by parse_options and will not appear in the resulting options dict.
        - If multiple kwargs normalize to the same camelCase key, the later one wins (overwrites earlier).

## Returns:

    - As a constructor, this method does not return a value. It always returns None implicitly.

## Raises:

    - Any exception raised by folium.utilities.parse_options (commonly AttributeError or TypeError if a key is not a valid string for camelization) will propagate. No exceptions are caught in this constructor.
